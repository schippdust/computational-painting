# App Architecture

## Drawing Philosophy

This app produces large, high-resolution generative images via **constructed complexity**: rather than rendering a live 3D scene, a small population of physics-driven agents ("vehicles", typically 1,000–3,000 active at a time) moves through 3D world space. Their projected 2D positions mark a persistent p5.js canvas. Marks accumulate frame by frame — the canvas is a growing record, not a real-time view.

This is why p5.js is chosen over three.js: the output is a 2D canvas that retains everything ever drawn. Three.js redraws the scene each frame from GPU state and cannot cheaply represent unlimited accumulating detail. Here, visual density is bounded only by time, not memory.

Canvas default size: **4600 × 4600 px**. Agents can occupy a world space of tens of thousands of units.

## Tech Stack

| Layer | Library |
|-------|---------|
| UI framework | Vue 3 (Composition API, `<script setup>`) |
| Language | TypeScript ~5.6 |
| Build | Vite 5 |
| State | Pinia |
| Canvas / simulation | p5.js 1.x (instance mode only) |
| UI components | Vuetify 3 |
| Math / matrices | mathjs (CoordinateSystem basis matrices) |
| Routing | unplugin-vue-router (file-based, `src/pages/`) |
| Component auto-import | unplugin-vue-components |
| API auto-import | unplugin-auto-import (Vue Composition API globals) |

## Folder Structure

```
src/
├── pages/           File-based routes. index.vue gates on appStore.initialized.
├── layouts/         Vue layout wrappers (default.vue)
├── components/      p5 canvas components + UI inputs. Each canvas component is a standalone sketch.
├── stores/          Pinia stores. app.ts is the primary store.
├── classes/         All simulation/rendering logic — see classes.md
├── plugins/         Vuetify + plugin registration
├── router/          Auto-generated by unplugin-vue-router
└── styles/          Global SCSS (Vuetify overrides)
```

## Data Flow

```
Vue onMounted
  └─ new P5(sketch, canvasElement)         instance mode sketch attached to DOM element
       ├─ p5.setup()
       │    ├─ create canvas (canvasWidth/Height from Pinia store)
       │    ├─ instantiate Geometry (Sphere, Line, Circle)
       │    ├─ instantiate Vehicle collections (BranchingCollection, etc.)
       │    ├─ instantiate WindSystem
       │    └─ instantiate Renderer (DotRenderer, SphereRenderer, etc.)
       └─ p5.draw()  [every frame]
            ├─ apply forces to vehicles (wind, steering, persistent forces)
            ├─ collection.update()  →  vehicle.update() × N  →  canvas marks persist
            ├─ renderer.renderVehicles(vehicles)  →  p5.ellipse / p5.line calls
            └─ update reactive refs (numberOfVehicles, numberOfFrames) for Vue UI
```

## Pinia Store (`src/stores/app.ts`)

The store is the only shared state between Vue and p5. Canvas components read from it at sketch setup time; they do not reactively re-run when store values change (p5 is imperative). Key fields:

- `canvasWidth` / `canvasHeight` — used at `p5.createCanvas()` time
- `camera: Camera3D` — shared camera; renderers use it for 3D→2D projection
- `pauseCanvas` — toggled by spacebar via `pressSpaceToPause(p5)` utility
- `initialized` — gates page routing between canvas view and setup inputs
